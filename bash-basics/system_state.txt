Процессы:
-------------------------------------------------------

Атрибуды процесса:
	- pid, ppid
	- состояния R-running, S-sleeping(waiting), D - блокирующий ввод/вывод (общение с диском), Z-zombie
	- nice приоритет на распределение проц. времени

Информация:
ps, читает информацию из procfs

Потребляемые ресурсы:
	-cpu
	-память

Сбор информации о системе:
-------------------------------------------------------
top - вывод PID/USER/PR(приоритет при приоретизации)/NIce/VIRT/RES/SHR/S/...
load average: текущее количество запущенных процессов (1 и меньше - система не занята)
		4 на 4х ядерной системе м.б. признаком того что система хорошо нагружена.
		3 числа 1 мин, 5 мин, 15 мин
состояния:
running
sleeping
stopped (ctrl+z) состояние T (trace)
zombie

процессорное время:
us - user time (выполнение программы)
id - idle time (простой)
wa - io wait ожидание ввода вывода более 10% это уже подозрительно (тупит в диске)
st - system time (переключение контекстов и системные вызовы)
ni - nice выполнение процессов с измененным приоритетом
hi, si - hardware,software interrupt
st - както связано с виртуализацией

кнопочки top:
>d - delay
>c - показать command
>H - показывать threads
>M - отсортировть по памяти
>P - отсортировать по процессору
>i - скрыть процессы которые ничег не делают

Другие команды (иногда требуется доустановить соответствующий пакет):
dstat - полезна для оценки состояния за интервал времени (выводит по строчкам) сеть, процессор и др.
iostat - статистика ввода вывода в системе
mpstat

df - свободное мето на дисках
df -i использование inod'ов
df -h человекочитаемо
mount - смонтированные фс тип, откуда, куда и с какими опциями
lscpu - проц
lspci - устройства pci
dmidecode
